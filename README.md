:pencil2: git相关
<br>
1. 提交规范
<br>
  ①每次提交必须写清楚相关message 
  <br>
  ②message字典：
  <br>
  + feat：新功能（feature）
  <br>
  + fix：修补bug
  <br>
  + docs：文档（documentation)
  <br>
  + style： 格式（不影响代码运行的变动）
  <br>
  + test：增加测试
  <br>
  + refactor：重构（即不是新增功能，也不是修改bug的代码变动）
  <br>
  + other：其他未分类
  <br>
  <br>
2. 提交命令
<br>
  git commit -m "type-<otherMessage>"
    <br>
  git push origin localbranch : remotebranch
